### 7. Add a reusable local build script

Create:

```text
scripts/build-docs-site.sh
```

Make it executable.

Behavior:

1. Install/build Doks from `docs/` only.
2. Generate Dokka HTML using the chosen Gradle Dokka task.
3. Copy Dokka output into `docs/api/`.
4. Build the Doks/Hugo site into `docs/public/`.
5. Copy `docs/api/` into `docs/public/api/`.
6. Create `docs/public/.nojekyll`.
7. Fail fast if either Doks output or Dokka output is missing.

Suggested script shape:

```bash
#!/usr/bin/env bash
set -euo pipefail

ROOT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")/.." && pwd)"
cd "$ROOT_DIR"

DOKKA_TASK="${DOKKA_TASK:-:dokkaGenerate}"
DOKKA_OUTPUT_DIR="${DOKKA_OUTPUT_DIR:-build/dokka/html}"
BASE_URL="${BASE_URL:-/lucene-kmp/}"

if [ ! -d docs ]; then
  echo "docs/ does not exist." >&2
  exit 1
fi

if [ ! -f docs/package.json ]; then
  echo "docs/package.json is missing. Doks/Thulite site is not set up correctly." >&2
  exit 1
fi

./gradlew "$DOKKA_TASK"

if [ ! -f "$DOKKA_OUTPUT_DIR/index.html" ]; then
  echo "Dokka output not found at $DOKKA_OUTPUT_DIR/index.html" >&2
  echo "Searching for generated Dokka index.html files..." >&2
  find . -path '*/build/dokka/*/index.html' -not -path './docs/node_modules/*' -not -path './docs/public/*' -print >&2 || true
  exit 1
fi

rm -rf docs/api
mkdir -p docs/api
cp -R "$DOKKA_OUTPUT_DIR"/. docs/api/

cd "$ROOT_DIR/docs"
npm run build -- --baseURL "$BASE_URL"
cd "$ROOT_DIR"

rm -rf docs/public/api
mkdir -p docs/public/api
cp -R docs/api/. docs/public/api/
touch docs/public/.nojekyll

if [ ! -f docs/public/index.html ]; then
  echo "docs/public/index.html is missing after site build." >&2
  exit 1
fi

if [ ! -f docs/api/index.html ]; then
  echo "docs/api/index.html is missing after Dokka copy." >&2
  exit 1
fi

if [ ! -f docs/public/api/index.html ]; then
  echo "docs/public/api/index.html is missing after Dokka copy into published site." >&2
  exit 1
fi

echo "Docs site built successfully: docs/public"
echo "Local Dokka API docs: docs/api"
echo "Published Dokka API docs: docs/public/api"
```

If repository inspection proves the actual Dokka output directory is different, update `DOKKA_OUTPUT_DIR` accordingly instead of relying on `find` as the primary path.

### 9. Create GitHub Actions workflow `.github/workflows/docs.yml`

Create:

```text
.github/workflows/docs.yml
```

Requirements:

- Build and deploy to GitHub Pages automatically on pushes that can affect the website or generated API docs.
- Include `docs/**` in path triggers because `docs/` is the website source directory and contains existing markdown pages.
- Include Kotlin source and Gradle build files in path triggers because generated Dokka output changes when public API or KDoc changes.
- Include manual `workflow_dispatch`.
- Use official GitHub Pages artifact deployment.
- Do not commit generated files.
- Build Doks/Hugo and Dokka in CI, then upload `docs/public` as the Pages artifact.
- Deploy only on `push` and `workflow_dispatch`, not on pull requests.

Important note about “detecting generated Dokka docs changes”:

Generated Dokka HTML should not be committed, so GitHub Actions cannot path-filter on generated HTML changes unless those files are committed. Instead, trigger the workflow on the inputs that change generated Dokka output: Kotlin source/KDoc, Gradle/Dokka config, README/license if they are included in docs, and the docs workflow/build script.

Use this as the starting workflow, then adjust versions to match repo conventions if needed:

```yaml
      - main
      - master
    paths:
      - 'docs/**'
      - '**/src/**/*.kt'
      - '**/src/**/*.kts'

      - 'gradle.properties'

      - 'gradlew'

      - 'docs/**'

      - '**/src/**/*.kts'
      - '**/build.gradle.kts'
      - 'build.gradle.kts'
      - 'settings.gradle.kts'
      - 'gradle.properties'
      - 'gradle/**'
      - 'gradlew'
      - 'README.md'
      - 'LICENSE*'
      - '.github/workflows/docs.yml'
      - 'scripts/build-docs-site.sh'
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: pages
  cancel-in-progress: false

defaults:
  run:
    shell: bash

jobs:
  build:
    name: Build documentation site
    runs-on: ubuntu-latest
    env:
      # Codex: inspect local `hugo version` and pin a compatible extended Hugo version here.
      # Doks/Thulite may require Hugo extended and Dart Sass.
      HUGO_VERSION: 0.163.3
      NODE_VERSION: 24
      # Codex: change this if the repo toolchain/CI uses 23 instead of 24.
      JAVA_VERSION: 24
      TZ: Asia/Tokyo
      HUGO_ENVIRONMENT: production
      HUGO_ENV: production
    steps:
      - name: Checkout
        uses: actions/checkout@v5
        with:
          submodules: recursive
          fetch-depth: 0

      - name: Set up JDK
        uses: actions/setup-java@v5
        with:
          distribution: temurin
          java-version: ${{ env.JAVA_VERSION }}

      - name: Set up Gradle
        uses: gradle/actions/setup-gradle@v4

        uses: actions/setup-node@v5

          node-version: ${{ env.NODE_VERSION }}
          cache: npm
          cache-dependency-path: docs/package-lock.json

      - name: Configure GitHub Pages
        id: pages
        uses: actions/configure-pages@v5

      - name: Install Hugo extended
        run: |
          curl -sL -o "${{ runner.temp }}/hugo.tar.gz" \
            "https://github.com/gohugoio/hugo/releases/download/v${HUGO_VERSION}/hugo_extended_${HUGO_VERSION}_linux-amd64.tar.gz"
          mkdir -p "${HOME}/.local/hugo"
          tar -C "${HOME}/.local/hugo" -xzf "${{ runner.temp }}/hugo.tar.gz"
          echo "${HOME}/.local/hugo" >> "${GITHUB_PATH}"

        run: sudo snap install dart-sass

      - name: Verify tool versions
        run: |
          java -version
          ./gradlew --version
          node --version
          npm --version
          hugo version
          sass --version

      - name: Install website dependencies
        working-directory: docs
        run: npm ci

      - name: Build Doks site and Dokka API docs
        env:
          BASE_URL: ${{ steps.pages.outputs.base_url }}/
          DOKKA_TASK: :dokkaGenerate
          DOKKA_OUTPUT_DIR: build/dokka/html
        run: scripts/build-docs-site.sh

      - name: Upload GitHub Pages artifact
        if: github.event_name == 'push' || github.event_name == 'workflow_dispatch'
        uses: actions/upload-pages-artifact@v4
        with:
          path: docs/public

  deploy:
    name: Deploy to GitHub Pages
    if: github.event_name == 'push' || github.event_name == 'workflow_dispatch'
    needs: build
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    permissions:
      pages: write
      id-token: write
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

If current GitHub docs or the repo standardize on newer action versions, update the major versions consistently.
